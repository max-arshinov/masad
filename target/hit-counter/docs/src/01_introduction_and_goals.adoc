== Introduction and Goals

- ✓Спроектировать счетчик посещений для сайта. Посещение - любое событие, происходящее на сайте (переход, нажатие кнопки, и тд).
- ✓Считать общее количество посетителей, количество уникальных посетителей в разрезах: по часам, дням, неделям, месяцам, по гео положению, по источнику трафика, браузеру, устройству, по utm меткам.
- ✓Отчеты и графики должны отдаваться не более чем за 1.5 с по любым комбинациям параметров за период не превышающий 3 месяца.

=== Ограничения:
- Система должна иметь возможность обслуживать все сайты рунета (примерно 100млрд событий в сутки).
- Допустимы задержки в 50-100 минут от действия пользователя до его отображения в статистике.
- Данные по сайтам должны храниться “вечно”* 

Комментарий: цифры, указанные в задании, нуждаются в валидации. Возможно, стейкхолдер сформулировал их неадекватно.

=== Requirements Overview

https://www.systeminterview.com/q=chatsystem&c=loggedin&v=v3&l=long is the original URL. Your service creates an alias with shorter length: https://tinyurl.com/y7keocwj. If you click the alias, it redirects you to the original URL.



=== Quality Goals

- *QG1:* 1 million new users per year, each user shortens an average of 100 links per year.
- *QG2:* The shortened links must be stored for 3 years.
- *QG3:* Redirection to the target address should occur in no more than 0.2 seconds.
- *QG4:* Click statistics must be tracked.
- *QG5:* System releases must occur seamlessly.
- *QG6:* The average link receives 10 visits per day.

*The failure of any two nodes should degrade system performance by no more than 0.1 seconds.*
